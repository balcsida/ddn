Cloud DB 2.0

The successor to the previous cloud database, this one is much more versatile and extensible.

Found bugs? Have feature requests? Report them here.

On the right, you can see the active connectors; These are the available database servers in which you can create and import databases.

Also works with .zip, .tar, .gz and .tar.gz archives

Known issues:

  • Uploading large compressed archives to the mysql servers fail if exploded archive is larger than 9 Gb (5G in case of .tar.gz)
  • Oracle import log not accessible yet
  • User/database name autogeneration not unique enough
